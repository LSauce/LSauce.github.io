Git（读音为/gɪt/）是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理

安装git

安装包：官网 蓝奏云
(Windons需下载,MacOS自带)

默认设置安装即可，相关设置可查看官方文档

使用git clone ssh地址 可下载库到本地
如：git clone git@github.com:symless/synergy-core.git

配置git

配置SSH key

  • 在任意目录 右键-Git Bash Here进入git终端
  • 输入指令生成SSH秘钥
    ssh-keygen -t rsa -C "邮件地址"// 填写你的github注册邮箱
  • 找到SSH秘钥生成地址(默认：user\.shh\)
    复制id_rsa.pub的文本（使用文本格式打开）
  • 打开Github，进入 Setting -> SSH and GPGkeys -> NewSSh Key
    粘贴id_ras.pub中文本，标题可以任意,保存秘钥
  • 查看是否成功
    ssh -T git@github.com #邮箱不用更改

配置git信息

  • 配置用户：
    git config --global user.name "github用户名"
    git config -- global user.email "github注册邮箱"

使用git将代码托管到Githubs

在Github上创建 repository

Repository name //仓库命名
Description //对功能进行描述
public //开源，所有人都能访问
private //闭源，私密仓库
initialize this repository with a README //创建一个说明文档
Add.gitgnore //选择仓库主要使用语言
Add a license //增加开源证书，可保留著作权

复制 Clone or download 处 SSH 地址

上传代码至Github

  • 在项目目录中，进入本地git终端，初始化库
    git init

  • 克隆项目至本地
    git clone ssh地址

  • 将要上传的文件放入本地库目录中

  • git终端进入本地库目录(cd命令)

  • 将变化加载至暂存区

    git add .

    git add -A //提交所有变化(git add -all的缩写)

    git add -u //提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)

    git add . //提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件

  • 说明这次提交，描述提交目的
    git commit -m "message"

  • 将本地库推送到托管服务器中

    git push -u origin master

    上面命令将本地的master分支推送到origin主机，加了参数-u后，以后即可直接用git push 代替git push origin master

    如果本地库中文件不是最新
    可用pull将远程master分支最新内容拉下来后与当前本地分支直接合并

    如：git pull origin master
